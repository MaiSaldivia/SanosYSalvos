# Contribución

## Estrategia de ramas

Usar Gitflow simplificado:

- `main`: código listo para producción/entrega
- `develop`: rama de integración
- `feature/nombre-funcionalidad`: trabajo nuevo
- `hotfix/nombre-error`: corrección urgente

## Reglas

1. Crear una rama por tarea.
2. Hacer commits pequeños y descriptivos.
3. Abrir Pull Request hacia `develop`.
4. Revisar compilación y pruebas antes de mergear.

## Convención de commits

- `feat:` nueva funcionalidad
- `fix:` corrección
- `docs:` documentación
- `refactor:` refactorización sin cambio funcional
- `chore:` tareas de mantenimiento
